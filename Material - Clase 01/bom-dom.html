Clase 01

BOM vs DOM

Web API (API Web)

Una interfaz de programación de aplicaciones (API por sus siglas en inglés), es un conjunto de rutinas que provee acceso a funciones de un determinado software.
Cuando trabajamos en Programación Web nos vamos a encontrar con dos tipos distintos de API's el primero son aquellas que nos van a permitir interactuar con los distintos elementos del navegador; mientras que el segundo son las API's que nos permiten obtener y manipular datos desde una fuente externa (API REST).
JavaScript ofrece una extensa variedad de API's para trabajar pero hay dos que son comunes a todos los proyectos: Window y Document

Browser Object Model (API Window)

El BOM es un objeto global (también conocido como objeto window) que contiene todas las API's con las que podemos trabajar. Además, a menos que especifiquemos lo contrario, se va a encargar de almacenar todas la variables, funciones y elementos que vayamos creando a lo largo de nuestra aplicación.
El BOM es un objeto tan global que no necesitamos referirnos a él de forma explícita cuando lo usamos.
El objeto global está presente en todos los entornos de trabajo por lo que podríamos acceder al mismo si usáramos Node.js para trabajar con JS desde el servidor. En este caso, usaríamos la palabra reservada 'global'. En actualizaciones recientes, se introdujo 'globalThis' para hacer referencia al objeto global en un proyecto que trabaje con más de un contexto de ejecución.

Document Object Model (API document)

Dentro del BOM nos encontramos con el DOM. El DOM es la API encargada de manejar el documento HTML.
El DOM genera una representación lógica del documento HTML conocida como DOM Tree. Esta representación se compone por un conjunto de nodos (objetos), relacionados entre sí que nos permite, desde JavaScript, interactuar con el documento HTML
Al momento de generar contenido dinámico (es decir, agregar nodos desde JS) hay que respetar una regla fundamental del DOM “Se deben minimizar al máximo la cantidad de cambios en el DOM”
