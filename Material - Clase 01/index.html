ECMAScript 2015

A lo largo de la existencia de Javascript y desde que se le encargó a ECMA el trabajo de su estandarización, han aparecido diversas versiones de Javascript con cada vez mayores funcionalidades, estructuras de control, etc. Esas son las versiones del estándar y hoy de la que nos ocupamos es de ES6, la sexta versión.
ES5 estuvo con nosotros durante muchos años y a día de hoy es la versión de Javascript más extendida en todo tipo de plataformas. Cuando alguien dice que conoce o usa Javascript es común entender que lo que usa es ES5, el Javascript con mayor índice de compatibilidad. De ésto también se entiende que, cuando queremos escribir un código compatible con todos los navegadores o sistemas, lo normal es que ese código sea ES5.
Para facilitar nuestra vida y poder comenzar a usar ES6 en cualquier proyecto, han surgido los transpiladores, una herramienta que ha venido a nuestro kit de desarrollo para quedarse.
Los transpiladores son programas capaces de traducir el código de un lenguaje para otro, o de una versión para otra. Por ejemplo, el código escrito en ES6, traducirlo a ES5. Dicho de otra manera, el código con posibles problemas de compatibilidad, hacerlo compatible con cualquier plataforma.

Variables

El primer gran cambio introducido en este standard es la posibilidad de declarar variables acotadas al ámbito en que fueron declaradas (let) para evitar que se sobre-escriba una variable que no debería. Otra cosa a tener en cuenta, es que al usar let las variables ya no se convierten de forma automática en propiedades del objeto global. Además, se introdujo la posibilidad de declarar variables que no pueden cambiar su contenido una vez asignadas (const). Por convención, las constantes se declaran con mayúsculas

Tipos de Datos

Los Tipos de Datos son estructuras básicas de la programación que nos permiten generar representaciones de la realidad y, de esa forma, moldear los programas según necesitemos. Todos los Lenguajes de Programación cuentan con su propio set de Tipos de Datos.
Los Tipos de Datos se dividen en 2 categorías: Primitivos y Abstractos.

  • Primitivos - Son Tipos de Datos proporcionados por el Lenguaje y permiten construir Tipos de Datos Abstractos y estructuras de datos:
    • Boolean - Valor lógico (true/false)
    • Number - Valor numérico (1, 1.5, -2, etc)
    • BigInt - Valores numéricos grandes (32434545n, 2324n)
    • String - Valor alfanumérico ("hola", "1", "a", "1 dato", etc)
    • Undefined - Ausencia de Tipo de Dato y valor
    • Null - Tipo de Dato sin valor
  • Abstractos - Son Tipos de Datos creados por el desarrollador en base a los Tipos de Datos Primitivos y otros Tipos de DAtos Abstractos. Sirven para representar de forma más fiel la realidad que el programa se propone emular o automatizar. En JS, los Tipos de Datos Abstractos se crean a partir del Tipo de Dato Object, es decir que todos los Tipos de Datos Abstractos son Objetos.
    • Objetos Planos(diccionarios) - Conjunto de datos organizados mediante el binomio clave: valor
    • Arrays(listas) - Conjunto de datos organizados de forma secuencial mediante un valor de índice
    • Funciones - Tipo de Dato que contiene código que se ejecuta según necesidad. Representan acciones

Funciones

Una función es un bloque de código, autocontenido, que se puede definir una vez y ejecutar en cualquier momento. Opcionalmente, puede aceptar parámetros y valores.
En Javascript, las funciones son objetos. Un tipo especial de objeto.
Se dice que las funciones son ciudadanos de primera clase porque pueden asignarse a un valor, pasarse como argumentos y usarse como un valor de retorno.
A partir de ECMAScript 6, es posible asignarle a las funciones valores por defecto igualando el nombre otorgado al parámetro al valor que deseamos se utilice en caso de no recibir un argumento en el momento de la ejecución.

Contexto de ejecución (this)

En JavaScript es importante prestar atención a dónde se declara una variable porque eso va a indicar si la misma es accesible. Esto se debe a lo que conocemos como 'ámbito de variable'. Cuando trabajamos con funciones tenemos que prestar atención, además, a lo que se conoce como 'contexto de ejecución' (this).
El contexto de ejecución (this) de una función nos va a indicar los distintos elementos que podemos afectar con la misma.
Todos los objetos en Javascript crean un contexto de ejecución (this) propio y todo lo que hacemos en JavaScript se encuentra almacenado en el objeto Window, por lo tanto el contexto de ejecución (this) por defecto es el objeto Window.
Las funciones declaradas(incluso las anónimas), trabajan sobre el contexto de ejecución(this) en el que fueron creadas. Las arrow function, en cambio, se saltan ese contexto y trabajan con el contexto al que pertenece el elemento que las contiene.

Arrow function

Una de las razones por las que se introdujeron las funciones flecha fue para eliminar complejidades del ámbito (this) y hacer que la ejecución de funciones sea mucho más intuitiva.

En las funciones tradicionales de manera predeterminada this está en el ámbito de window

Las funciones flecha no predeterminan this al ámbito o alcance de window, más bien se ejecutan en el ámbito o alcance en que se crean

Template String

Los template string nos permiten escribir de forma más amigable un string dentro de una variable que combine el contenido de una o varias porciones de string con variables u elementos propios de JS

Nuevas formas de iterar con For

Métodos de Array

Map

.map() es un método iterador similar a .forEach() con una diferencia fundamental que nos va a indicar en qué momento utilizamos uno u otro.
Al trabajar con .forEach() tenemos la posibilidad de ejecutar una función callback para cada elemento iterado, pero si quisiéramos hacer un return dentro de dicha callback su valor siempre será null. En otras palabras, con .forEach() no podemos retornar valores.
Para aquellos casos en que la iteración de los elementos almacenados en un array requieran el retorno del valor de los mismos contamos con .map().
El método map() crea un nuevo array con los resultados de la llamada a la función indicada aplicados a cada uno de sus elementos.

Filter

Recorre el array y retorna un nuevo array con aquellos elementos que pasen una determinada condición.

Find

Recorre el array y retorna la primera coincidencia del elemento que se busca.

Includes

Determina si un array incluye un determinado elemento y retorna un booleano según corresponda.
