Programación Orientada a Eventos

La Programación orientada a eventos es un paradigma de programación en el que la estructura y la ejecución de los programas van determinados por los sucesos o acciones que ocurren en el sistema, definidos por el usuario o por el propio sistema.
Como desarrolladores, nuestro objetivo es preparar el código para que cuando ocurra un evento con ciertas características, se ejecute un código asociado.

Cómo trabajr con Eventos

Mediante HTML

Consiste en agregar el evento a un elemento HTML mediante el uso de atributos HTML. Esta opción no es particularmente recomendada porque, entre otras cosas, implica mezclar lenguaje JavaScript con HTML

Mediante propiedades

En esta opción, desde JavaScript, tomamos el elemento al que queremos agregar el evento y lo asociamos utilizando el prefijo on

API Web - EventTarget (Listeners)

EventTarget es una interfaz implementada por los objetos que pueden administrar eventos y sus escuchadores. EventTarget nos permite agregar escuchas (addEventListener) de un evento a un elemento y, por lo tanto, generar la ejecución de varias funciones callback sobre un sólo elemento. Con EventTarget también vamos a poder eliminar una escucha (removeEventTarget) cuando esta ya no sea necesaria.

Remover un listener implica que la callback pasada en el removedor deje de ejecutarse, pero esto no impide la ejecución de otras callbacks

Objeto Event

En algunos casos, nos puede interesar profundizar en detalles relacionados con el evento. Por ejemplo, si estamos escuchando un evento de click de ratón, nos puede interesar saber con que botón del ratón se ha hecho click, o en que punto concreto de la pantalla. Estos detalles son posibles obtenerlos a través del objeto event.
Para acceder al objeto event, sólo necesitamos agregar un parámetro a la función callback del listener

Pasar Parámetros

A una callback de evento sólo se puede pasar el parámetro que hace referencia al objeto evento por lo que, de necesitar parametros debemos ejecutar una función dentro de la callback que reciba los parámetros necesarios

Propagación

Para entendender el principio de propagación primero debemos tener en claro cómo se relacionan los distintos elementos (conocidos como nodos) del DOM. El DOM nace en la creación del Objeto Document y, a partir de ahí se genera el conocido como Árbol del DOM que es un mapeo de las distintas relaciones de los nodos

Un Evento tiene 3 fases:

  1. Event Capturing - Se busca al elemento que desato desde el primer elemento del Árbol del DOM)
  2. Event Target - Se llega al elemento que desato el evento
  3. Event Bubbling - Se vuelve desde el elemento que desato el evento hacia el primer elemento del Árbol del DOM

El principio de propagación nos permite manejar cómo se va a alcanzar al elemento que desató el evento. Casi todos los eventos se propagan

Propagación

Cuando un evento ocurre en un elemento, este primero ejecuta los manejadores que tiene asignados, luego los manejadores de su padre, y así hasta otros ancestros.

Detener la propagación

Una propagación de evento empieza desde el elemento objetivo hacia arriba. Normalmente este continúa hasta la etiqueta y luego hacia el objeto document, algunos eventos incluso alcanzan window, llamando a todos los manejadores en el camino.
Pero cualquier manejador podría decidir que el evento se ha procesado por completo y detener su propagación.
El método para esto es event.stopPropagation().

Delegación de eventos

La captura y el propagación nos permiten implementar uno de los más poderosos patrones de manejo de eventos llamado delegación de eventos.
La idea es que si tenemos muchos elementos manejados de manera similar podemos, en lugar de asignar un manejador a cada uno de ellos, poner un único manejador a su ancestro común.
Esta práctica, además de ayudarnos a mejorar la performance de nuestra aplicación, se vuelve necesaria cuando trabajamos con elementos creados de forma dinámica.
