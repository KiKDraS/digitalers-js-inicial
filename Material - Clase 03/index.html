Clase 03

Expresiones Regulares

Las expresiones regulares son patrones que proporcionan una potente forma de buscar y reemplazar texto. En JavaScript cobran especial relevancia porque, además, nos permiten generar validaciones.
Están disponibles a través del objeto RegExp y se encuentran integrados en los métodos de String.
Hay dos sintaxis que se pueden usar para crear un objeto de expresión regular.

Banderas

Las banderas son elementos que afectan a la búsqueda.
En JavaScript sólo hay 6:

  • i - La búsqueda no distingue entre mayúsculas y minúsculas: no hay diferencia entre A y a
  • g - La búsqueda encuentra todas las coincidencias. Sin ella sólo se devuelve la primera
  • m - Modo multilínea. Sólo afecta al comportamiento de anclas (^,$). Coinciden no solo al principio y al final de la cadena, sino también al inicio/final de la línea.
  • s - Habilita el modo “dotall”, que permite que un punto . coincida con el carácter de línea nueva \n
  • u - Permite el soporte completo de Unicode. La bandera permite el procesamiento correcto de pares sustitutos.
  • y - Modo “adhesivo”: búsqueda en la posición exacta del texto

Clases de caracteres

Una clase de caracteres es una notación especial que coincide con cualquier símbolo de un determinado conjunto.
Para cada clase de caracteres existe una “clase inversa”, denotada con la misma letra, pero en mayúscula. El “inverso” significa que coincide con todos los demás caracteres

Anclas: inicio ^ y final $ de la cadena

Las anclas ^ y $ son pruebas. Ellas tienen ancho cero.
En otras palabras, no coinciden con un carácter, sino que obligan al motor regexp a verificar la condición (inicio/fin de texto).
El patrón ^Mary significa: “inicio de cadena y luego Mary”.
Similar a esto, podemos probar si la cadena termina con nieve usando nieve$
Ambos anclajes ^...$ se usan juntos a menudo para probar si una cadena coincide completamente con el patrón.

Límite de palabra: \b

Un límite de palabra \b es una prueba, al igual que ^ y $.
Cuando el motor regex (módulo de programa que implementa la búsqueda de expresiones regulares) se encuentra con \b, comprueba que la posición en la cadena es un límite de palabra.
La prueba de límite de palabra \b verifica que debe haber un \w en un lado de la posición y "no \w"- en el otro lado.
Pero \w significa una letra latina a-z (o un dígito o un guion bajo), por lo que la prueba no funciona para otros caracteres, p.ej.: letras cirílicas o jeroglíficos.
Hay tres posiciones diferentes que califican como límites de palabras:

  • Al comienzo de la cadena, si el primer carácter de cadena es un carácter de palabra \w.
  • Entre dos caracteres en la cadena, donde uno es un carácter de palabra \w y el otro no.
  • Al final de la cadena, si el último carácter de la cadena es un carácter de palabra \w.

Escapar caracteres especiales

Como hemos visto, una barra invertida \ se usa para denotar clases de caracteres, p.ej. \d. Por lo tanto, es un carácter especial en expresiones regulares (al igual que en las cadenas regulares).
También hay otros caracteres especiales que tienen un significado especial en una expresión regular, tales como [ ] { } ( ) \ ^ $ . | ? * +
Para usar un carácter especial como uno normal, hay que agregar una barra invertida adelante. Por ej: '.' busca cualquier carácter posible; mientras que '\.' busca un punto.

Conjuntos y Rangos

Tanto los conjuntos como los rangos se representan mediante []. La diferencia consiste en que los conjuntos (ej. [oi]), sólo retornaran true si se encuentra o u i; mientras que en los rangos (ej. [0-5]) retornara true si se encuentra un dígito entre el 0 y el 5.
Al igual que con las clases, es posible usar los rangos y conjuntos para escapar caracteres. Por ej. [^aeyo] retornará true con cualquier carácter excepto a, e, y u o.
Al trabajar con conjuntos, también es posible utilizarlos para encontrar dos palabras que comportan caracteres. Por ej.: /Java(Script)?/ encontrará las palabras 'Java' y 'JavaScript'

Cuantificadores +, *, ? y {n}

Al momento de realizar una búsqueda de caracteres podría interesarnos limitar las coincidencias a una cantidad específica. Para eso utilizamos los cuantificadores:

  • {n} - Agregando un número entre llaves a nuestra búsqueda podemos especificar la cantidad.
    • El recuento exacto: {x}
      \d{5} Denota exactamente 5 dígitos, igual que \d\d\d\d\d.
    • El rango: {x,y}, coincide x-y veces
      Para encontrar números de 3 a 5 dígitos, podemos poner los límites en llaves: \d{3,5}
    • Podemos omitir el límite superior
      Luego, una regexp \d{3,} busca secuencias de dígitos de longitud 3 o más
  • Abreviaciones
    • + => 'uno o más' ({1,})
    • ? => 'cero o uno' ({0,1})
    • * => 'cero o más' ({0,})

Grupos de Captura

Al incluir una parte del patrón entre paréntesis, estamos creando un grupo de captura.
Un grupo de captura permite 2 posibles acciones:

  1. Obtener una parte de la coincidencia como un elemento separado en la matriz de resultados.
  2. Al colocar un cuantificador después del paréntesis, se aplica a los paréntesis en su conjunto

Lookahead

Es utilizado para analizar el los valores ingresados en una RegExp y retornar un true sólo si las condiciones planteadas en el Lookahead se cumplen

Métodos de RegExp y String

match()

El método str.match(regexp) encuentra coincidencias para las expresiones regulares (regexp) en la cadena (str).
Tiene 3 modos:

  1. Si la expresión regular (regexp) no tiene la bandera g, retorna un array con los grupos capturados y las propiedades index (posición de la coincidencia), input (cadena de entrada, igual a str)
  2. Si la expresión regular (regexp) tiene la bandera g, retorna un array de todas las coincidencias como cadenas, sin capturar grupos y otros detalles.
  3. Si no hay coincidencias, no importa si tiene la bandera g o no, retorna null

split()

Divide la cadena usando la expresión regular (o una sub-cadena) como delimitador.

search()

El método str.search(regexp) retorna la posición de la primera coincidencia o -1 si no encuentra nada. Solamente retorna la primer coincidencia.

regexp.test()

El método regexp.test(str) busca por una coincidencia y retorna true/false si existe.

Si la expresión regular tiene la bandera g, el método regexp.test busca la propiedad regexp.lastIndex y la actualiza.
Entonces podemos usarlo para buscar desde un posición dada

Validación de formularios

El trabajo con formularios, tan importante a la hora de comunicarnos con el cliente, genera ciertas vulnerabilidades que, de no ser atendidas, podrían permitir ataques maliciosos.
Existen varias técnicas, principalmente desde el back-end, para impedir la recepción de código malicioso pero desde JavaScript podemos aportar para mejorar la seguridad. Para eso, vamos a hacer uso de algunos métodos dados del lenguaje.

  • trim() - Elimina los caracteres en blanco al principio y al final del string
  • includes(x) - Nos permite pasar un carácter o cadena de caracteres para revisar si el string determinado lo contiene. Devuelve un booleano
  • indexOf(x) - Símil includes, pero devuelve la posición del elemento buscado en el string o -1 en caso de que no exista
